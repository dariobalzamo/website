E-commerce di videogiochi

L’obiettivo dell’applicativo è quello di simulare la gestione di uno store di videogames, effettuando le operazioni d i manipolazione e visualizzazione dei prodotti. L'e-commerce di videogiochi è stato sviluppato con un database esterno, utilizzando come DBMS Mysql. Inoltre, la web app sfrutta la tecnologia delle Jsp/Servlet. È stato adottato un pattern architetturale MVC ed aggiunto un package per la classe Singleton Connettore stabilendo una connessione tra l’applicativo web Java e il db esterno sfruttando la libreria JDBC. Il webserver utilizzato è quello di Apache Tomcat. Al fine di consentire un'interazione client-server ci siamo avvalsi delle JSP (Servlet Just in time) definendo l'interfaccia utente, mentre lato back-end abbiamo usato una classe figlia di HttpServlet per gestire a livello logico le richieste di tipo get (mandando i dati direttamente dalla querystring) o di tipo post (compilando un form).

Gestionale aziendale Spring Rest

L'obiettivo di tale progetto è di testare, attraverso il tool Postman, il nostro applicativo web API-Rest in modo semplice e veloce, effettuando chiamate HTTP in GET, POST, PUT e DELETE.
L'applicativo consente di gestire i dipendenti di un'azienda, effettuando le operazioni di CRUD. La web-app è stata sviluppata con il framework Spring e sfruttando la tecnologia Hibernate sfruttando l'ORM. Con Hibernate Ho auto-generato le tabelle di un db relazionale (storage) ed applicato il crud sfruttando il Dependency Injection (DI) con l'annotazione @Autowired; questo mi ha permesso di ridurre notevolmente i tempi di sviluppo. Inoltre, una volta creato un controller (annotato opportunamente con @Controller), ogni metodo dichiarato all'interno (e annotato con @RequestMapping).
